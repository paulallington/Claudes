              Identifies this device's exported sessions inside the sync folder. Defaults to the machine's hostname.
            

            

              
              

                
                
              

              A local folder synced to the cloud by your tool of choice (Google Drive Desktop, OneDrive, iCloud, Syncthing…). Claudes mirrors your project sessions in/out of here. We never talk to a cloud API directly.
            

            

              
              
            

            

            

              How to use:

              1. Set the path above on each device (pointing to the same cloud-synced folder).

              2. Right-click a project in the sidebar → "Sync convos" to start uploading that project's chats.

              3. On the other device, right-click the matching project → "Import syncs…" and pick this device's folder under the sync source.

              4. Resumed sessions show up in /resume on either machine.

    Keyboard shortcuts
    
          
    macOS users: ⌘ works wherever Ctrl is shown below.
    
          
    
            
              
    Ctrl+KCommand palette — fuzzy-search projects, slash commands, app actions
    
              
    Ctrl+PQuick-open file — fuzzy search files in the active project, opens in inline editor
    
              
    Ctrl+FFind in terminal (when a column is focused) — Enter / Shift+Enter cycles matches
    
              
    Ctrl+Shift+FSearch sessions — toggle between transcripts and your prompt history
    
              
    Ctrl+Shift+GSearch file contents project-wide (respects .gitignore)
    
              
    Ctrl+Shift+TSpawn a new column in the active project
    
              
    Ctrl+Shift+RAdd a new row to the layout
    
              
    Ctrl+Shift+WKill the focused column
    
              
    Ctrl+Shift+EToggle Explorer panel
    
              
    Ctrl+Shift+MMaximize / restore the focused column
    
              
    Ctrl+← / Ctrl+→Navigate between columns
    
              
    Ctrl+1…9Jump to column N
    
              
    Ctrl+BToggle the Projects sidebar
    
              
    Ctrl++ / Ctrl+- / Ctrl+0Change terminal font size
    
              
    ?Open this shortcuts reference
    
              
    EscClose the active modal / popover
    
                
          
    
        
    

        
    
          
    Plan limits & usage
    
          
      
            
    • Sidebar mini bar — live Session and Week percentages from the OAuth-authed plan-limits endpoint. Click to open the full Usage modal.
      • 
            
    • Threshold notifications — system notification + window flash when 70% / 90% are crossed on session or weekly windows. Toggle individually in Settings.
      • 
            
    • Auto-pause prompt — at 90% weekly, optional confirm dialog to disable global automations until you re-enable them.
      • 
            
    • Session attribution Δ pill — each Claude column header shows how much of the 5-hour session quota that column has consumed since spawn. Hover for snapshot vs current numbers.
      • 
            
    • Context-window meter — per-column live tokens / model-max indicator (e.g. 12k/200k). Amber at 70%, red at 90% — compact proactively before auto-compact.
      • 
            
    • Cost dashboard — Usage modal → Cost tab. Total + per-model + per-project + 30-day chart, with Today / 7-day / 30-day / All-time filter.
      • 
          
    
        
    

        
    
          
    Workflow speed
    
          
      
            
    • Cross-column broadcast (⤳) — send the same prompt to multiple Claude columns at once. Picks up custom column titles and project names automatically.
      • 
            
    • Session full-text search (🔎 / Ctrl+Shift+F) — grep across every session JSONL on disk. Click a hit to switch project and resume that session.
      • 
            
    • My prompts search — same modal, "My prompts" radio mode. Searches ~/.claude/history.jsonl. Click a hit to copy the prompt to your clipboard.
      • 
            
    • Cmd-K command palette — fuzzy search across projects (switch / spawn), slash commands (sent to focused column), and app actions (open Usage, kill column, respawn, etc.).
      • 
          
    
        
    

        
    
          
    Editor & explorer
    
          
      
            
    • Clickable file paths in terminal — patterns like src/foo.js:42 become clickable in any column. Clicks open the inline editor focused on that line. URLs also auto-link.
      • 
            
    • Drag-into-terminal — drag a file from the Explorer tree onto a terminal; the path (quoted if it has spaces) is written to the pty.
      • 
            
    • Syntax preview — inline file editor → Preview button switches to a Prism-highlighted read view; toggle back to edit.
      • 
            
    • Open in external editor — project context menu / Explorer right-click sends the path to your code command (configurable in Settings → Tools).
      • 
            
    • Quick open (Ctrl+P) — fuzzy file pickers across the active project; Ctrl+Shift+G for content grep.
      • 
            
    • Auto-refresh — Explorer watches the project root and updates when files change on disk.
      • 
          
    
        
    

        
    
          
    Git tab
    
          
      
            
    • Merge / rebase — branch dropdown shows per-branch merge and rebase buttons. Conflict banner above the changes list lets you resolve files (use ours / use theirs / open) and continue / abort the operation.
      • 
            
    • File history & blame — right-click any file in the changes list. History panel includes one-click Cherry-pick onto current.
      • 
          
    
        
    

        
    
          
    Project extensions
    
          
      
            
    • MCP servers — project context menu → Manage MCP servers… for a CRUD UI over .mcp.json.
      • 
            
    • Sub-agents / skills / commands — manage .claude/agents/, .claude/skills/, .claude/commands/ in both project and global scope from one modal.
      • 
            
    • Named layouts — save and restore row/column shapes per project from the context menu.
      • 
            
    • Auto-detected run configs — Run tab picks up package.json scripts, Makefile targets, Cargo bins, and pyproject.toml scripts.
      • 
          
    
        
    

        
    
          
    Snippets & observability
    
          
      
            
    • Prompt snippets (📝) — save reusable prompt templates with {{variable}} placeholders. Type \trigger + Enter in any Claude column to expand; the app prompts you for each variable, then erases the typed trigger and sends the expanded body.
      • 
            
    • Hook event inspector — Hooks tab in the explorer panel streams every PreToolUse, PostToolUse, etc. event from Claude CLI. Filter, pause, click to expand JSON. Click Connect hooks to auto-configure ~/.claude/settings.json.
      • 
          
    
        
    

        
    
          
    Reliability & automation
    
          
      
            
    • Endpoint failover — optional Fallback endpoint on each preset. If a Claude column exits within 5 seconds with a non-zero code, the app respawns it on the fallback endpoint with a ↺ failover badge in the header.
      • 
            
    • Automation usage gates — per-agent Skip if session usage above N% and Skip if weekly usage above N% fields. Scheduled runs auto-skip when over the threshold so they don't burn the rest of your quota.
